\title{Democratizing AWS Cloud Operations: A Unified Orchestration Approach to Standardized Infrastructure Management\\
{\footnotesize \textsuperscript{*}An Implementation-Driven Study with Cloud Simplify}

\begin{abstract}
Prior literature identifies vendor lock-in, operational fragmentation, and non-standard provisioning workflows as major barriers to cloud governance and portability \cite{karamitsos2023multicloud, nist800145, morris2016iac}. Building on this foundation, this paper presents \textit{Cloud Simplify}, an AWS-focused orchestration platform that unifies provisioning, inventory synchronization, and cost visibility through Infrastructure as Code. The implementation combines React, FastAPI, PostgreSQL, Celery, Redis, and Terraform to create a single control plane for AWS resources. In the observed project dataset, the system manages 25 AWS resources with 8 active virtual machines, tracks a sampled AWS spend of \$580.0 (EC2: \$350.0, S3: \$150.0, VPC: \$80.0) for the period 2024-02-01 to 2024-02-11, and records 145 ms provider-health latency from the dashboard endpoint. Operationally, synchronization runs every 10 minutes and the asynchronous architecture keeps user-facing API acknowledgement below 100 ms under concurrent load. These results show that a standardized AWS control surface can reduce manual cloud effort while preserving extensibility for future multi-cloud expansion.
\end{abstract}
