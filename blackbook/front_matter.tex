% ---------------- ABSTRACT ----------------
\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}

The transition from traditional on-premise infrastructure to cloud computing has been revolutionary, providing unparalleled scalability and cost-efficiency. However, as organizations grow, the reliance on a single cloud vendor often leads to ``vendor lock-in,'' escalating costs, and single-point-of-failure risks. This has prompted a strategic shift toward Multi-Cloud architectures.

This project introduces \textbf{Nebula: The Multi-Cloud Command Center}, a high-performance orchestration engine designed to unify infrastructure management across Amazon Web Services (AWS), Microsoft Azure, and Google Cloud Platform (GCP). Nebula abstracts these providers into a single control plane, leveraging Infrastructure as Code (IaC) through Terraform and an asynchronous execution engine powered by FastAPI and Celery. 

Key innovations include an \textbf{AI-driven Troubleshooting Copilot} that parses Terraform execution logs to provide deterministic remediation steps, a \textbf{Security Vault} for encrypted credential management, and \textbf{Real-time Inventory Syncing}. The resulting platform empowers developers to provision, monitor, and troubleshoot resources across different clouds without switching between disparate consoles.
