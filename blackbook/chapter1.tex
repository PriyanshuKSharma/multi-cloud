\chapter{INTRODUCTION}

The "Cloud First" era has transformed the global digital economy. As organizations move from simple cloud adoption to "Multi-Cloud Strategic Maturity," the need for unified management platforms has become a critical operational requirement.

\section{Historical Context of Computing}
Modern computing has undergone several paradigms of transformation, each significantly increasing the abstraction between the user and the underlying hardware. Understanding the evolution of these paradigms is critical to appreciating the value proposition of the Nebula Multi-Cloud Command Center. 

\subsection{The Mainframe Era (1950s - 1970s)}
In the early days of computing, massive mainframe computers occupied entire rooms. These systems were centralized, and users accessed them via "dumb terminals." While powerful for their time, they were prohibitively expensive, difficult to scale, and represented a single point of failure. Management was strictly manual and required a specialized workforce.

\subsection{Client-Server Architecture (1980s - 1990s)}
The rise of the Personal Computer (PC) led to the client-server model. Computing tasks were distributed between a server (providing data or services) and a client (requesting them). While more flexible than mainframes, this era introduced the "silo" problem, where different departments managed their own physical servers, leading to underutilization of hardware resources and high maintenance costs.

\subsection{The Virtualization Revolution (2000s)}
Virtualization transformed the relationship between software and hardware. Hypervisors allowed multiple Virtual Machines (VMs) to run on a single physical server. This was the technological precursor to cloud computing, enabling efficient resource utilization and the ability to "spin up" servers in minutes rather than weeks.

\section{The Rise of Cloud Computing}
Cloud computing is the logical culmination of virtualization, broad network access, and automated management. The National Institute of Standards and Technology (NIST) defines cloud computing as a model for enabling ubiquitous, convenient, on-demand network access to a shared pool of configurable computing resources.

\subsection{NIST Service Models}
To understand the scope of the Nebula platform, we must categorize the services it manages:
\begin{enumerate}
    \item \textbf{Infrastructure as Service (IaaS)}: Provides fundamental computing resources such as Virtual Machines (AWS EC2, Azure VM), storage, and networks. Nebula provides deep integration at this layer.
    \item \textbf{Platform as Service (PaaS)}: Provides a platform allowing customers to develop, run, and manage applications without the complexity of infrastructure. Examples include AWS Lambda and Azure Functions, which Nebula manages as "Network Functions."
    \item \textbf{Software as Service (SaaS)}: Software is licensed on a subscription basis and is centrally hosted. While Nebula is a SaaS-like platform itself, it manages the underlying IaaS and PaaS to support enterprise SaaS deployments.
\end{enumerate}

\subsection{NIST Deployment Models}
\begin{itemize}
    \item \textbf{Public Cloud}: Resources are owned and operated by a third-party service provider (AWS, Azure, GCP).
    \item \textbf{Private Cloud}: Resources used exclusively by a single business or organization.
    \item \textbf{Hybrid Cloud}: A composition of two or more clouds (private, community, or public) that remain unique entities but are bound together.
    \item \textbf{Multi-Cloud}: The use of multiple public cloud services from different providers. Nebula is specifically designed to dominate this space.
\end{itemize}

\section{The Multi-Cloud Strategic Shift}
Organizations are increasingly adopting a multi-cloud strategy to avoid "Vendor Lock-in." However, this shift introduces significant management friction. 

\subsection{Drivers for Multi-Cloud Adoption}
\begin{itemize}
    \item \textbf{Best-of-Breed Services}: Using GCP for data analytics while keeping enterprise applications on Azure.
    \item \textbf{Data Sovereignty}: Storing data in specific geographical regions to comply with GDPR or local regulations.
    \item \textbf{Negotiation Leverage}: Having infrastructure on multiple clouds prevents reliance on a single vendor's pricing model.
\end{itemize}

\section{Problem Statement and Motivation}
Despite its benefits, multi-cloud management is fragmented. Currently, an administrator must:
\begin{enumerate}
    \item Manage high-security credentials across three different portals.
    \item Learn separate naming conventions (e.g., Security Groups in AWS vs. NSG in Azure).
    \item Handle long-running deployments that often time out in the browser.
\end{enumerate}
The motivation for \textbf{Nebula} is to create a "Unified Control Plane" that abstracts these complexities into a single, high-performance API and User Interface.

\section{Project Objectives and Scope}
The specific objectives of this project are:
\begin{itemize}
    \item Achieve 100% visibility of cloud inventory across providers.
    \item Reduce troubleshooting time using LLM-based log analysis.
    \item Standardize resource provisioning workflows.
    \item implementing a secure credential vault using application-layer encryption.
    \item Enable real-time deployment log streaming using WebSockets or periodic polling.
\end{itemize}
The scope covers the initial development of the core engine, security vault, and a React-based interactive cockpit.
