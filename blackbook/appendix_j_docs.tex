\subsection{Dashboard Request Flow Example}
\textbf{Source:} \texttt{docs/README.md}
\begin{lstlisting}[numbers=none,basicstyle=\ttfamily\tiny,caption={Dashboard Request Flow Adapted from docs/README.md}]
1. User opens browser -> http://localhost:5173
2. React app loads -> Dashboard component mounts
3. useQuery hook triggers -> api.get('/dashboard/stats')
4. Axios sends GET /dashboard/stats with JWT token
5. FastAPI verifies JWT, extracts user
6. Route handler queries ResourceInventory and CostData
7. SQLAlchemy executes SQL against PostgreSQL
8. PostgreSQL returns results
9. FastAPI formats JSON response
10. Response sent to frontend
11. React Query caches data
12. Dashboard re-renders with real metrics and charts
\end{lstlisting}

\subsection{Background Resource Sync Flow}
\textbf{Source:} \texttt{docs/SUMMARY.md}
\begin{lstlisting}[numbers=none,basicstyle=\ttfamily\tiny,caption={Background Resource Sync Flow Adapted from docs/SUMMARY.md}]
1. Celery Beat triggers every 10 minutes
2. sync_all_users_resources() starts
3. Query all users with credentials
4. For each user enqueue sync_user_resources(user_id)
5. For each credential:
   - If AWS: sync_aws_resources()
   - If Azure: sync_azure_resources()
   - If GCP: sync_gcp_resources()
6. Sync task executes:
   a. Decrypt credentials
   b. Initialize cloud SDK client
   c. Call provider API
   d. Transform to ResourceInventory model
   e. Upsert to database
   f. Update provider_health
7. Task completes and fresh data becomes available to the API
\end{lstlisting}

\subsection{Terraform Provisioning Flow}
\textbf{Source:} \texttt{docs/SUMMARY.md}
\begin{lstlisting}[numbers=none,basicstyle=\ttfamily\tiny,caption={Terraform Provisioning Flow Adapted from docs/SUMMARY.md}]
1. User submits POST /resources/ with configuration
2. Create Resource record with status='pending'
3. Enqueue provision_resource_task(resource_id)
4. Celery worker picks up task
5. Generate Terraform .tf file from config
6. Run terraform init
7. Run terraform plan
8. Run terraform apply -auto-approve
9. Parse outputs (instance_id, public_ip, etc.)
10. Update Resource record:
    - status='active'
    - terraform_output={...}
11. Store Terraform state in terraform_states
12. User polls GET /resources/{id} for status
\end{lstlisting}
