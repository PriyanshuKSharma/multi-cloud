\section{Failure Analysis Case Studies (AI Engine)}
A critical metric for Nebula is the accuracy of its failure analysis engine. We documented three specific cases.

\subsection{Case Study 1: Missing IAM Permissions}
When attempting to provision an S3 bucket in a restricted region, Terraform returned an "Access Denied" error. The AI Copilot scrubbed the log and correctly identified that the IAM user lacked the \texttt{s3:CreateBucket} permission in the specified VPC.

\subsection{Case Study 2: Provider Quota Limits}
During a bulk provisioning test (TC-12), Azure returned a "Quota Exceeded" status for VM cores. The AI engine interpreted this correctly and suggested that the user request a quota increase from the Azure portal, providing a direct link to the support page.
